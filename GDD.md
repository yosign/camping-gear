# 像素专注 GDD

> 版本 0.1 · 2026-03-05 · Draft

## 目标体验（One Sentence）

**「和像素小人一起工作，每次专注都是一次小冒险。」**

核心情感：沉浸感 + 陪伴感。用户不是在「坚持专注」，而是在「舍不得离开这个世界」。

---

## 目标用户

**玩家画像：小林**

- 25 岁，设计师/学生/自由职业者
- 喜欢像素复古游戏美学，常听 lo-fi 工作
- 用过 Forest / 番茄钟，觉得冷硬没感情
- 她不缺「知道要专注」，缺的是「愿意待在专注状态里」的动力
- 手机里装着 Animal Crossing，喜欢每天回去看小动物

**玩家类型（Bartle）：** 主要面向 Socializer（陪伴感）+ Explorer（新场景/新故事）

**乐趣类型（Lazzaro）：**
- **Sensation**：美的像素场景 + 音乐
- **Fantasy**：我和小人一起在做同一件事

---

## 核心循环（Core Loop）

> **v0.2 核心玩法变更**：专注时间 → 抽卡次数 → 抽宠物/装备。金币系统废弃，改为抽卡票系统。

### 30 秒循环 — 「进入仪式」

必须在没有任何奖励的情况下也令人满足。

```
用户选择场景 →
  [音效] 像素门/窗打开，小人走进来
  [动画] 场景从模糊到清晰（聚焦隐喻）
  [音乐] 背景音乐淡入

用户点击「开始专注」→
  [动画] 小人停下来看向用户，转身坐下开始工作
  [视觉] 屏幕边缘出现微弱像素光晕（专注状态标志）
  [文案] 「再专注 25 分钟，就能抽一次！」（当前票数 + 进度）
```

**设计原则：** 开始这个动作本身必须有仪式感和满足感。

### 30 分钟循环 — 「一次专注 Session」

**结构：积累 → 等待 → 获得抽卡票**

| 阶段 | 内容 |
|---|---|
| 开始 | 显示当前抽卡票数量 + 本次专注能获得几票的进度 |
| 专注中 | 场景动态变化（天气推移、时间流逝）；小人偶有小动作 |
| 结束 | 获得抽卡票，动画提示「+1 票」，引导去抽卡 |

**结束动作的 Juice：**
```
计时结束（25 分钟）→
  [音效] 轻柔钟声
  [动画] 小人伸懒腰，转向用户
  [视觉] 画面中央漂浮出一张金色抽卡票
  [文案] 「你专注了 25 分钟，获得 1 张抽卡券！」
  [按钮] 「去抽卡」/ 「继续专注」
```

**专注完成后不自动抽卡——用户主动点击才抽**，拉的瞬间才是爽感高峰。

### 30 小时循环 — 「角色养成」

**核心驱动：持续专注 → 攒票 → 抽卡 → 收集 + 情感附着**

```
专注时间 → 抽卡票 → 抽宠物 / 装备 → 角色个性化 → 舍不得离开
```

关键设计：**小人和宠物共同成长**。宠物陪在小人旁边，用户看到的不是单个小人，而是「自己攒出来的一个小世界」。

这是「收集型」+ 「表达型」双重乐趣——每只宠物都是某段专注时间的具象化。

## 三种专注模式

### 陪伴模式（默认）
- 轻专注，小人始终在屏
- 适合需要环境感的日常工作
- 时间不限，随时开始/结束

### 探索模式（核心功能）
- 深度专注，设定目标时长
- 开始时小人「出发」，结束时「揭晓」
- AI 生成冒险故事，结果每次不同
- 核心的 Variable Ratio 奖励机制

### ~~回忆模式~~（MVP 不做）
> 已移除。会稀释「专注 = 在 app 里」的核心行为绑定。后续版本根据数据决定是否加入。

## 场景与音乐

**MVP 场景数量：2 个（精，不是多）**

每个场景包含：
- 高质量像素背景（静态 + 动态层）
- 场景专属背景音乐（lo-fi 风格）
- 动态天气/时间系统：白天 → 黄昏 → 夜晚，晴天 → 阴天 → 下雨
- 对应环境音效变化

候选场景（待定 2 个）：
1. 雨天书房（温暖室内，窗外雨景）
2. 深夜咖啡厅（城市感，暖光）

## 抽卡系统

### 抽卡动画（核心爽感）

```
用户点击「抽卡」→
  [动画] 抽卡票漂浮到卡池中央，发光旋转
  [音效] 像素魔法音效
  [动画] 光效爆开，宠物/装备从中浮现
  [结果] 宠物/装备名称 + 稀有度光晕 + 展示动画
  [按钮] 「放入场景」/ 「装备到小人」/ 「继续抽」
```

十连时：连续出10张牌面，最后一张翻牌有停顿感。

### 抽卡池内容

两个独立池，共用同一张抽卡票：

**宠物池（Pet Pool）**

每只宠物由三个维度组合生成，保证每只都有唯一性：

| 维度 | 选项 |
|---|---|
| 物种（Species） | 猫、兔子、乌龟、鸭子、龙、幽灵、蘑菇、水豚、章鱼、机器人等 18 种 |
| 眼睛（Eye） | ·  ✦  ×  ◉  @  °（6 种） |
| 帽子（Hat） | 无 / 皇冠 / 礼帽 / 螺旋桨 / 光环 / 巫师帽 / 毛线帽（7 种） |

物种决定稀有度（如龙 = rare，幽灵 = epic，传说物种 = legendary）。

**装备池（Equipment Pool）**

小人的服装配件，纯外观，不影响任何数值：
- 帽子系列（草帽 / 猫耳 / 头盔...）
- 上衣系列（毛衣 / 工装 / 校服...）
- 配件系列（围巾 / 眼镜 / 背包...）

## 小人与宠物系统

### 宠物陪伴机制

- 宠物显示在**小人旁边**，有独立的 idle 动画
- 专注时：宠物也在「工作」（偶尔打盹、翻书、发呆）
- 用户可设置当前展示宠物（1只上场，其余在图鉴里）
- 不同场景天气下，宠物有对应小反应（下雨时猫咪躲到小人脚边）

### 装备机制

- 抽到装备直接装到小人身上，立刻可见
- 用户可随时切换已拥有的装备
- 装备是纯视觉，不影响抽卡概率或任何数值

### 小人记忆系统（保留）

- 每次专注自动记录：日期、时长、场景、同行宠物、装备状态
- 小人「日记」可查看历史
- 用户翻看：「那天我戴着猫耳帽，旁边坐着那只幽灵，专注了 3 小时」

---

## 收藏图鉴

- 所有抽到的宠物进入图鉴，按物种 + 稀有度排列
- 未解锁的显示剪影（制造收集欲）
- 每只宠物记录「获得时间」+ 「当时的专注时长」
- Shiny 变体有专属标记

**重复宠物处理**：同物种 + 同眼睛 + 同帽子的宠物再次获得 → 自动转化为「星尘」，可用于指定兑换稀有宠物（50 星尘换 rare，150 换 epic）。

---

## 摄像头功能（精简版）

**定位：关心，不是监督**

| 功能 | 实现方式 | 禁止 |
|---|---|---|
| 护眼提示 | 检测长时间未眨眼 → 温和提醒休息 | 不做「你离开了」的提醒 |
| 欢迎回来 | 检测到用户回到镜头 → 小人做欢迎动作 | 不做「你走了多久」的记录 |

**MVP 决策：摄像头功能后置。** 先验证核心假设（美学驱动专注），摄像头在 v1.1 加入。

## 经济系统

> **v0.2 重构**：废弃金币体系，改为抽卡票机制。

### 抽卡票获取（Faucet）

| 专注模式 | 获取规则 |
|---|---|
| 陪伴模式 | 每 25 分钟 1 张票 |
| 探索模式 | 每 25 分钟 1 张票 + 完整完成额外 +1 张 |

**设计原则：** 用户每天 2 次 25 分钟专注 = 2 张基础票，探索模式再 +2 = 4 张 / 天。攒 10 张触发十连保底，约 2-3 天攒够一次十连。节奏感适中。

### 抽卡票消耗（Sink）

- 单抽：1 张票
- 十连：10 张票（保底必出稀有及以上）

### 保底机制

| 条件 | 效果 |
|---|---|
| 每 10 抽 | 必出 rare 及以上 |
| 每 50 抽 | 必出 epic 及以上 |
| 每 100 抽 | 必出 legendary |

保底计数跨 session 累计，不会重置。

### 稀有度与概率

| 稀有度 | 概率 | 视觉表现 |
|---|---|---|
| ⬜ Common | 60% | 静态，无特效 |
| 🟩 Uncommon | 25% | 轻微色彩变化 |
| 🟦 Rare | 10% | 发光动效，蓝色光晕 |
| 🟪 Epic | 4% | 粒子特效，紫色 |
| 🌟 Legendary | 1% | 专属动画，金色 + 场景特效 |

**Shiny 变体**：每种宠物有极低概率（~0.5%）出现特殊配色版本，视觉独特，无额外功能。

## 第一次使用体验（FTUE）

遵循「不靠文字教程，靠设计引导」原则：

```
1. 打开 app → 直接进入默认场景（雨天书房）
   → 小人坐在桌前，旁边有只默认小猫陪着（common 品质，预设）

2. 画面底部出现唯一按钮：「开始专注」
   → 点击后直接进入陪伴模式，同时显示「再 25 分钟可抽卡」

3. 新用户特权：直接获得 3 张抽卡票
   → 引导立刻体验一次抽卡
   → 完整播放抽卡动画，让用户感受到核心钩子

4. 抽卡后，宠物出现在场景里陪着小人
   → 用户立刻看到「我攒出来的小伙伴」的效果

5. 展示抽卡票进度条
   → 「专注 25 分钟再抽一次」，引导第一次真实专注
```

**教程测试：** 去掉所有文字，用户是否能在 3 分钟内开始第一次专注？若不能，重新设计流程而非加文字。

## 设计风险 & 待验证假设

| 假设 | 验证方式 | 风险等级 |
|---|---|---|
| 美学场景能显著降低切屏频率 | A/B 测 vs 无场景版本，对比切屏次数 | 🔴 核心假设 |
| 目标用户愿意为「更美的体验」付费 | TestFlight 小范围测试 + 问卷 | 🔴 核心假设 |
| 揭晓机制能驱动用户完成完整 session | 对比探索模式 vs 陪伴模式的完成率 | 🟡 重要 |
| 小人养成的沉没成本足够留存 | 30 天留存率，对比 Forest | 🟡 重要 |
| 2 个场景足够支撑早期用户 | 新增场景前的用户流失节点分析 | 🟢 次要 |

---

## 成功指标

| 指标 | 目标值 | 测量方式 |
|---|---|---|
| Session 平均时长 | ≥ 25 分钟 | App 内埋点 |
| 探索模式完成率 | ≥ 70% | 开始 vs 完成对比 |
| Day 7 留存 | ≥ 40% | 留存曲线 |
| Day 30 留存 | ≥ 20% | 留存曲线 |
| 付费转化率 | ≥ 8% | 内购数据 |
| 切屏频率 | 低于 Forest 对照组 30% | 屏幕时间 API |

## MVP 功能范围

**做：**
- 陪伴模式 + 探索模式
- 2 个高质量场景（含动态天气/时间）
- 抽卡票系统（25分钟=1票）
- 抽卡动画（单抽 + 十连）
- 宠物系统（物种 × 眼睛 × 帽子组合生成，MVP 先做 6 种物种）
- 装备系统（小人服装，MVP 先做 10 件）
- 保底机制（10 抽必 rare）
- 收藏图鉴（含未解锁剪影）
- 小人记忆/日记

**不做（v1.1 考虑）：**
- 回忆模式
- 摄像头功能
- 3 个以上场景
- 全部 18 种物种（MVP 6 种，后续版本陆续解锁）
- 社交/晒宠物功能
- 照片生成小人
- Shiny 变体（MVP 后加）
